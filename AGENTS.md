## Tooling
- On this Windows Codex setup, ripgrep is installed; use `rg` and `rg --files` as the default text and file search tools.
- Fall back to native PowerShell search only if `rg` is unavailable in the active environment.

## Subagents
- Use subagents only when the user explicitly asks for them or the task clearly has independent scopes worth delegating.
- Split work by ownership: separate files, modules, services, screens, or responsibilities.
- Keep the immediate blocking task in the main session; delegate sidecar exploration, review, or disjoint implementation that can run in parallel.
- Do not have multiple worker agents edit the same files or shared generated outputs at the same time.
- For broad reviews, use mapper/research/reviewer roles first, then an integration review before handoff when changes were parallel.
- For broad implementation, assign workers only when each write set is isolated and the prompt includes context, owned files, forbidden files, expected output, and verification commands.

## Frontend Work
- For any frontend page or screen, use `image-guided-frontend` and run the work image-first before implementation.
- If no suitable visual reference is provided, generate one before coding unless the user explicitly opts out or image generation is unavailable.
- For seat maps, booking flows, floor plans, dashboards, admin pages, and other geometry- or data-constrained UI, still start image-first, but verify final correctness against structured data, schema, coordinates, permissions, and real product rules.

## Verification
- Prove the requested behavior directly first.
- Then run one adjacent regression check near the same route, API, state flow, serializer, or build path.
- If the likely verification command above is wrong or too slow, replace it with the fastest trusted repo-owned command.

## Risk Areas
- Learning progress is localStorage-only; changes to `types/progress.ts` or `lib/storage/progressStorage.ts` need migration-safe normalization.
- Lesson completion, quiz scoring, and exercise feedback define the user-visible learning contract.
- Tone.js audio playback requires browser interaction; code can be tested in Node, but real audio behavior needs browser proof.
- Generator fallback behavior can mislead learners if requested options and actual template choices are not shown clearly.
- Production release checks depend on `NEXT_PUBLIC_SITE_URL` and the smoke script.
